Política de Privacidad

Última actualización: 2026-02-26

Resumen: ColmenaPro está diseñada para funcionar en local. Por defecto, tus registros (colmenas, visitas, producción, genealogía) se guardan en tu dispositivo. El backend de ColmenaPro se utiliza solo para servicios online (por ejemplo, verificación de acceso PRO/Google Play y funcionalidades que requieran conexión, si las activas).

1. Responsable del tratamiento

Responsable: ColmenaPro (proyecto independiente).
Contacto: apicolaabouza@gmail.com

Este documento es un texto informativo y práctico para usuarios. Si necesitas una versión con datos fiscales/razón social, se puede completar cuando definas la entidad publicadora definitiva.

2. Qué datos maneja la app

ColmenaPro permite registrar información apícola introducida por el usuario, por ejemplo:

  • Colmenas y núcleos
  • Visitas, observaciones e incidencias
  • Producción (cosechas y cantidades)
  • Genealogía/linajes

ColmenaPro no requiere que introduzcas nombre, dirección o datos personales para funcionar en modo local.

3. Dónde se guardan los datos

Los datos se almacenan en el dispositivo. La app está pensada para poder usarse sin conexión. No se suben automáticamente a servidores externos.

4. Copias de seguridad (backup)

La app puede ofrecer funciones de exportación/restauración para copias de seguridad (por ejemplo, archivos de exportación/ZIP). Estas copias las gestiona el usuario (guardándolas en su almacenamiento, PC o nube personal).

ColmenaPro no realiza copias automáticas en servidores externos salvo que en el futuro actives una función online explícita y se actualice esta política.

5. Backend de ColmenaPro: qué es y cuándo interviene

ColmenaPro dispone de un servicio backend (API) para soportar funciones online. En la configuración actual, el backend se utiliza principalmente para:

  • Autenticación y autorización de acceso a servicios online (por ejemplo, verificación de acceso PRO).
  • Emisión de tokens de acceso (por ejemplo, JWT) para llamadas autenticadas a la API.
  • Verificación con Google Play cuando sea necesario (por ejemplo, validación de compra/suscripción y/o integridad de instalación).

Si no usas funciones online, la app puede funcionar en modo local sin necesidad de comunicarse con el backend.

6. Datos técnicos tratados por el backend

Cuando se usa el backend, se pueden tratar datos técnicos mínimos para poder prestar el servicio:

  • Identificador técnico de instalación (por ejemplo, un ID generado por la app para identificar la instalación).
  • Tokens de sesión/acceso (por ejemplo, JWT) y datos asociados a su validez (caducidad, plan/entitlements).
  • Metadatos de verificación Google Play (por ejemplo, resultados de verificación relacionados con Play Billing / Play Integrity si se usa esa verificación).
  • Datos de red: dirección IP y logs técnicos necesarios para seguridad, prevención de abuso y diagnóstico.

El objetivo es seguridad, control de acceso y operación del servicio. No se realiza seguimiento comercial del usuario.

7. Contenido apícola y servicios online

El contenido apícola (colmenas, visitas, genealogía, producción) permanece en local por defecto. Si en el futuro activas o se habilitan funciones de sincronización/consulta online (por ejemplo, modo online de genealogía), esta política se actualizará para describir:

  • Qué datos exactos se envían
  • Con qué finalidad
  • Base legal
  • Plazos de conservación

8. Google Play y servicios de terceros

ColmenaPro se distribuirá a través de Google Play. Google puede tratar datos técnicos necesarios para descarga, actualización, facturación (si hay suscripciones) y seguridad, conforme a sus políticas.

ColmenaPro no vende datos personales ni comparte tus registros apícolas con terceros para publicidad.

9. Base legal (GDPR)

  • Ejecución del servicio (art. 6.1.b GDPR): para operar funciones online cuando las uses (autenticación/PRO).
  • Interés legítimo (art. 6.1.f GDPR): seguridad, prevención de abuso, y registros técnicos mínimos.
  • Consentimiento (art. 6.1.a GDPR): si en el futuro se añadieran analíticas opcionales o funciones adicionales que lo requieran.

10. Conservación

En local: los datos permanecen en tu dispositivo hasta que los elimines o desinstales la app. En backend: los logs técnicos y tokens se conservan el tiempo estrictamente necesario para seguridad y operación (por ejemplo, para detectar abusos, resolver incidencias y mantener sesiones).

11. Seguridad

Se aplican medidas técnicas para proteger la API y las sesiones (por ejemplo, emisión de tokens, validaciones y controles de acceso). Aun así, ningún sistema es infalible y se recomienda mantener el dispositivo protegido (bloqueo de pantalla, copias de seguridad propias, etc.).

12. Derechos del usuario

Puedes solicitar información, rectificación o supresión de datos vinculados a servicios online (si existiesen datos identificables), y ejercer derechos de acceso/oposición/limitación según corresponda.
